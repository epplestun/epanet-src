Settings structure

There are three levels of settings which are form, field, and validators.

The FormValidation plugin can be called as following:

{% highlight javascript %} $(formSelector).formValidation({ // Indicate the framework // It can be: bootstrap, bootstrap4, foundation5, foundation, pure, semantic, uikit framework: 'bootstrap', // ... Form settings go here ... fields: { fieldName: { // ... Field settings go here ... validators: { specificValidatorName: { // ... common validator settings go here ... // ... specific validator settings ... } } } } }); {% endhighlight %}
If the field name contains special characters such as ., [, ], you must wrap it between single or double quote. See the Validating field with special name example

Most of settings can be set via HTML 5 attributes prefixed with data-fv.

For example, the following call:

{% highlight javascript %} $(formSelector).formValidation({ framework: 'bootstrap', excluded: [':disabled', ':hidden', ':not(:visible)'], icon: { valid: 'glyphicon glyphicon-ok', invalid: 'glyphicon glyphicon-remove', validating: 'glyphicon glyphicon-refresh' }, live: 'enabled', message: 'This value is not valid', trigger: null, fields: ... }); {% endhighlight %}

is equivalent with the one below:

{% highlight html %}
...
{% endhighlight %}
